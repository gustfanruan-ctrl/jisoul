## 1. 项目概览

JiSoul 是一个面向销售对话场景的智能辅助系统，当前已经形成以下核心能力：

- 后端基于 FastAPI，提供话术生成、知识库管理、增强检索等 API。
- 检索链路升级为“向量粗排 + 关键词/元数据增强 + 可选 reranker 精排”。
- 前端支持文本输入与实时语音识别（ASR）接入，能在同一交互面板中完成建议生成。
- 已补齐评测脚本与反馈链路，可用于线上/远程批量评估质量回归。

## 2. 本次迭代核心改动

### 前端

- 主交互页及状态管理更新：`App.tsx`、`TextInputPanel.tsx`、`FloatingAssistantPanel.tsx`、`store/index.ts`、`types/index.ts` 等。

## 4. 启动与运行（快速路径）

### 后端

```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

### 前端

```bash
cd frontend
npm install
npm run dev
```

## 5. 生产交接建议（优先级）

- **P0：密钥治理**  
  所有 API Key、Token、远程连接凭据统一迁移到新的运维账号与密钥系统，禁用旧个人密钥。
- **P0：评测基线固化**  
  保留并定时执行 RAG 评测脚本，沉淀“每周基线分”与回归报告。
- **P1：向量库与模型资产管理**  
  对 `backend/data/chroma` 建立备份与恢复策略；模型缓存目录统一规划。
- **P1：日志与告警**  
  建议接入请求耗时、召回命中率、fallback 率、ASR 成功率等核心指标监控。
- **P2：接口收敛**  
  根据业务选择保留基础版或增强版 suggestions 入口，减少双路径维护成本。

## 6. 风险与注意事项

- 当前仓库包含本地 Chroma 持久化数据文件，后续需明确是否持续纳入版本管理。
- 实时 ASR 与检索链路均依赖外部服务，需确认新维护人具备对应账号与网络权限。
- 评测脚本较多，建议统一入口脚本并补充 README，降低后续使用门槛。
